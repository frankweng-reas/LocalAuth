# 🎨 管理介面使用說明

## 訪問方式

開啟瀏覽器，訪問：

```
http://localhost:3000/admin
```

---

## 介面功能

### 1. 📊 統計區域

頁面頂部顯示兩個統計卡片：
- **總用戶數** - 系統中所有用戶的數量
- **活躍用戶** - `isActive = true` 的用戶數量

### 2. ➕ 新增用戶面板（左側）

**欄位說明：**
- **Email** *（必填）
  - 必須符合 email 格式
  - 不可重複
  
- **密碼** *（必填）
  - 最少 6 個字元
  - 使用 bcrypt 自動加密
  
- **名稱**（選填）
  - 可以留空

**操作步驟：**
1. 填寫表單
2. 點擊「建立用戶」按鈕
3. 看到成功訊息後，右側列表會自動更新

**錯誤處理：**
- Email 格式錯誤 → 顯示紅色錯誤訊息
- Email 已存在 → 顯示「Email already registered」
- 密碼太短 → 顯示「password must be longer than or equal to 6 characters」

### 3. 👥 用戶列表（右側）

**顯示資訊：**
- 名稱（或「未設定名稱」）
- 啟用狀態標籤（綠色/紅色）
- Email
- 用戶 ID
- 建立時間

**操作：**
- **刪除按鈕** - 點擊後會彈出確認對話框
  - 確認後永久刪除用戶
  - 刪除成功後列表自動更新

**自動更新：**
- 每 30 秒自動重新載入列表
- 手動重新整理頁面也會更新

## 使用範例

### 範例 1：新增第一個用戶

1. 在左側面板輸入：
   ```
   Email: admin@company.com
   密碼: admin123
   名稱: 系統管理員
   ```

2. 點擊「建立用戶」

3. 看到綠色成功訊息：
   ```
   ✅ 成功建立用戶: admin@company.com
   ```

4. 右側列表立即顯示新用戶

### 範例 2：批次新增多個用戶

重複以下步驟：
1. 填寫不同的 email
2. 點擊建立
3. 表單自動清空，繼續下一個

### 範例 3：查看用戶資料

在用戶卡片中可以看到：
```
系統管理員              [啟用]      [刪除]
📧 admin@company.com
ID: a1b2c3d4-...
📅 建立時間: 2026/2/11 下午9:00:46
```

### 範例 4：刪除測試用戶

1. 找到要刪除的用戶卡片
2. 點擊紅色「刪除」按鈕
3. 確認對話框出現：
   ```
   確定要刪除用戶 "test@example.com" 嗎？
   
   此操作無法復原。
   ```
4. 點擊「確定」
5. 看到成功訊息後，用戶從列表消失

## API 整合

管理介面使用以下 API 端點：

| 功能 | API | 方法 |
|------|-----|------|
| 新增用戶 | `/auth/register` | POST |
| 列出用戶 | `/users` | GET |
| 刪除用戶 | `/users/:id` | DELETE |

## 疑難排解

### 問題：頁面顯示「無法載入用戶列表」

**原因：** 後端服務未啟動

**解決：**
```bash
npm run start:dev
```

### 問題：新增用戶失敗

**常見原因：**
1. Email 格式錯誤
2. Email 已存在
3. 密碼太短（< 6 字元）

**檢查：** 查看紅色錯誤訊息的詳細說明

### 問題：刪除用戶後還在列表中

**原因：** 自動更新尚未執行

**解決：** 
- 等待 30 秒自動更新
- 或手動重新整理頁面（F5）

## 進階使用

### 在生產環境使用

**注意事項：**
1. 目前管理介面**沒有認證保護**
2. 建議在生產環境中：
   - 使用 HTTP Basic Auth
   - 或加入 JWT 認證
   - 或限制 IP 訪問

### 自訂管理介面

管理介面是單一 HTML 檔案：
```
public/index.html
```

你可以自由修改：
- 樣式（CSS 在 `<style>` 標籤中）
- 功能（JavaScript 在 `<script>` 標籤中）
- 新增更多功能（編輯用戶、角色管理等）

### API Base URL 配置

如果後端在不同的 port 或域名：

修改 `public/index.html` 中的：
```javascript
const API_BASE = 'http://localhost:3000';
```

改為：
```javascript
const API_BASE = 'http://your-api-domain.com';
```

## 建議的工作流程

### 開發階段
1. 啟動開發伺服器
2. 開啟管理介面
3. 在介面中新增測試用戶
4. 使用 API 或介面測試功能
5. 測試完畢後刪除測試用戶

### 部署階段
1. 加入認證保護（推薦）
2. 設定正確的 API_BASE
3. 考慮使用反向代理（nginx）
4. 限制管理介面訪問權限
